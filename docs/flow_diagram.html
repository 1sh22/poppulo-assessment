PDF RAG — End-to-End Flow one user query, start to finish

What happens between a user typing a question and the cited answer rendering on screen.

1. The browser
Step 1

User types a question

The chat input collects the question. Prior turns in this session are kept so follow-ups have context.

app/components/Chat.tsx
Step 2

POST to /api/query

The browser sends the question, recent history, and any cited chunk IDs as JSON, then opens a streaming reader.

fetch("/api/query")
2. Server entry point
Step 3

API route validates input

The route reads the JSON body, makes sure there is a real question, and prepares a streaming response (SSE).

app/api/query/route.ts
Step 4

Load doc registry

Pulls the list of available documents so the rewriter knows which papers exist and the response can be scoped to them.

lib/rag/documents.ts
3. Retrieval pipeline
Step 5

Conversational rewrite

If there is chat history, a small LLM call rewrites "tell me more" into a self-contained query and decides intent: rephrase, deepen, or new topic.

retrieve.ts → conversationalRewrite
Step 6

Query expansion

Another small LLM call writes 3 paraphrases of the question. Different wording → better recall against the paper's vocabulary.

retrieve.ts → expandQuery
Step 7

Dense search (per paraphrase)

Each paraphrase becomes an embedding vector. Qdrant returns the top 20 chunks by cosine similarity.

embed.ts + qdrant.ts
Step 8

Sparse search (per paraphrase)

Same paraphrases run through a BM25 keyword index in memory. Catches rare technical terms embeddings might miss.

bm25.ts
Step 9

Reciprocal Rank Fusion

All ranked lists merge into one fused ranking. No score calibration needed — chunks that appear high in many lists rise to the top.

rrf.ts
Step 10

LLM re-rank → top K

A small LLM scores each candidate 0–3 for relevance. The best 6 (default) survive and are sent to the answerer.

retrieve.ts → llmRerank
4. Answer generation
Step 11

Send retrieved chunks to client

Server immediately emits a "retrieved" SSE event so the UI can show the passages while the model is still thinking.

SSE event: type=retrieved
Step 12

Stream answer from gpt-4o-mini

System prompt requires bullet form, plain language, and a [n] citation marker on every bullet. Tokens stream back as deltas.

generate.ts → generateAnswerStreaming
Step 13

Derive citations from text

When the stream ends, the server scans the answer for [n] markers and links each one back to its source chunk (page + paragraph + quote).

generate.ts → deriveCitationsFromText
5. Browser renders
Step 14

Live token rendering

The chat component appends each delta into the assistant's message bubble. Markdown renders bullets and citation badges as they appear.

Chat.tsx → AnswerText
Step 15

Final "done" event

The server sends one last event with citation metadata and elapsed time. The chat shows clickable source chips beneath the answer.

SSE event: type=done
Step 16

Click citation → open PDF

Clicking [1] opens a side panel that loads the original PDF, jumps to the cited page, and highlights the quote in yellow.

PdfPreview.tsx + /api/pdf/:id
