Where each file lives, what it owns, and which other files it talks to. Color = layer.

UI Layer

app/page.tsx
Root page. Renders the Workbench shell.
→ Workbench
Workbench.tsx
3-pane layout: sidebar, chat, PDF viewer. Owns chat sessions in localStorage.
→ Chat, PdfPreview, AppSidebar
Chat.tsx
Question input, SSE reader, message rendering with inline citation badges.
→ /api/query, /api/ingest
PdfPreview.tsx
react-pdf viewer. Jumps to page and highlights the quoted span.
→ /api/pdf/:id
DocumentsPanel.tsx
Drag-drop upload + URL ingest + delete list. Used inside the sidebar.
→ /api/ingest, /api/documents
app-sidebar.tsx
Left sidebar: chat history list, documents list, new chat / delete dialogs.
→ Workbench callbacks
components/ui/*
shadcn primitives: Button, Dialog, Sidebar, Markdown, etc. Used everywhere.

API Routes

/api/ingest
POST a PDF (multipart) or { url }. Hands bytes to the ingest pipeline.
→ ingestPdf
/api/query
POST { question, history }. Streams retrieved chunks → answer tokens → done event over SSE.
→ retrieve, generate, documents
/api/documents
GET list of indexed documents.
→ listDocuments
/api/documents/[id]
DELETE a document. Removes from Qdrant, BM25, registry.
→ qdrant, ingest, documents
/api/pdf/[id]
GET raw PDF bytes (or redirect to Vercel Blob URL) for the viewer.
→ documents, storage
/api/debug
Diagnostic endpoint: shows Qdrant health, BM25 size, sample retrievals.

RAG Core (lib/rag/*)

pdf.ts
Parses PDF to (page, paragraph) text using pdfjs-dist. Strips headers/footers.
→ pdfjs-dist
chunk.ts
Sentence-aware chunker. ~450 tokens, ~80 overlap, never crosses paragraphs.
embed.ts
Batches text → OpenAI embeddings with retry/backoff.
→ OpenAI
bm25.ts
In-memory BM25 index, serializable to JSON. Tokenizer drops stopwords, keeps hyphens.
qdrant.ts
Thin wrapper over Qdrant client: collection bootstrap, upsert, search, delete by docId.
→ Qdrant Cloud
rrf.ts
Reciprocal Rank Fusion. Merges multiple ranked lists with no score calibration.
retrieve.ts
Top-level pipeline: rewrite → expand → dense+sparse → RRF → LLM rerank.
→ embed, qdrant, bm25, rrf, openai
generate.ts
Builds the cited-answer prompt, streams tokens, parses [n] markers into citations. Substring validator catches fabrications.
→ openai
ingest.ts
Orchestrates: parse → chunk → embed → upsert → BM25 update → store PDF → register doc.
→ pdf, chunk, embed, qdrant, bm25, storage, documents
storage.ts
Two-mode key/value store: Vercel Blob in prod, /.data on disk in dev.
documents.ts
Registry of indexed documents (id, name, page count, blob URL). Backed by storage.
→ storage
openai.ts
Singleton OpenAI client (caches across hot invocations).
types.ts
Shared TypeScript types: Chunk, Citation, ScoredChunk, DocumentMeta, etc.
index.ts
Barrel re-export of the public RAG surface.

External Services + Persistent Storage

OpenAI
Embeddings (text-embedding-3-small, 1536 dims) + chat completions (gpt-4o-mini for rewrite, rerank, answer).
Qdrant Cloud
Vector DB. Collection rag_chunks, cosine distance, payload index on docId for per-doc filtering.
Vercel Blob
Production storage for raw PDFs, BM25 JSON index, and document registry. Falls back to /.data on disk in dev.
.data/ (local)
Dev-only: PDFs in .data/pdfs/, BM25 in .data/indexes/bm25.json, registry in .data/registry/documents.json.

Key dependency lines

  • Chat.tsx → /api/query → retrieve.ts → embed.ts + qdrant.ts + bm25.ts → rrf.ts → generate.ts
  • DocumentsPanel.tsx / Chat.tsx → /api/ingest → ingest.ts → pdf.ts → chunk.ts → embed.ts → qdrant.ts + bm25.ts → storage.ts + documents.ts
  • PdfPreview.tsx → /api/pdf/[id] → documents.ts + storage.ts (or 302 to Vercel Blob CDN)
  • /api/documents/[id] DELETE → qdrant.deleteDocument + removeDocumentFromBm25 + removeDocument
  • scripts/seed.ts → ingest.ts (downloads two arXiv PDFs and runs the full ingestion).
